# 프로젝트 규칙 (CLAUDE.md)

국회직 8급 공무원 합격후기 검색 엔진 — `assembly8-review-finder`

## 핵심 원칙
- **전략**: 검색 API 오케스트레이션. 자체 웹 크롤러 금지(범위 외).
- **보안**: API 키는 서버(`.env`)에만 둔다. 프론트엔드/네트워크 응답/깃에 키를 절대 노출하지 않는다(`.env`는 `.gitignore`에 포함).
- **1차 소스는 네이버.** 보조 소스(서드파티 SERP, 레거시 Google CSE)는 어댑터 인터페이스(`SourceAdapter`) 뒤에 둔다(교체·plug-in 가능). v1 기본은 네이버 단독.
- 외부 API는 항상 모킹 가능한 어댑터로 감싼다. 직접 호출 코드를 여기저기 흩뿌리지 않는다.
- **쿼터 가드**: 일일 한도 초과 전에 차단(`QuotaGuard`). 캐시 우선(`TTLCache`).
- **저작권**: 후기 본문 전체를 저장·재배포하지 않는다. 제목 + 짧은 스니펫 + 원문 링크만 다룬다(검색 인덱싱 수준).

## 코드 규약
- 백엔드: Python 3.10+ / FastAPI / httpx(async). 도메인 로직은 `app/core`, 소스는 `app/adapters`.
- 프론트: Vite + Vanilla TS PWA. 프론트는 자체 백엔드의 `/api/*`만 호출한다.
- 외부 호출은 `respx`로 모킹해 쿼터 0 소모로 테스트한다. 새 기능엔 테스트를 동반한다.
- 커밋 단위는 작게. PR마다 리뷰(CodeRabbit 등)를 반영한 뒤 머지한다.

## 셸 호환성
- 모든 명령/스크립트는 **Windows PowerShell 5.1과 7.x 모두에서** 동작해야 한다.
- `&&`, `||`, `??`, 삼항 연산자를 사용하지 않는다. 줄바꿈 또는 `;`로 구분한다.
- 파일 저장 시 인코딩은 UTF-8.

## 인계
- 산출물 경로/포맷·인터페이스가 바뀌면 `HANDOFF.md`를 갱신한다.

## 확정된 결정 (지시서 14장)
1. 보조 소스: **(c) 네이버 단독 (v1)**. 폭 부족 시 어댑터로 SERP/CSE 추가.
2. 프론트 프레임워크: **Vanilla TS + Vite**.
3. 캐시: **인메모리 TTL** (`Cache` 프로토콜로 추후 SQLite 교체 가능).
4. 배포: **로컬 실행 + Docker** (단일 서버가 API와 정적 프론트를 함께 서빙).
